# Content Repurposing Cascade

Systematic approach to turning one hero piece into multiple derivative content pieces across formats and channels.

---

## The Cascade Model

One hero piece → multiple derivatives, each adapted for its channel's native format.

```
Hero Content (long-form)
├── Social derivatives (5-7 pieces)
│   ├── LinkedIn post (insight excerpt)
│   ├── LinkedIn carousel (key framework)
│   ├── Twitter/X thread (numbered takeaways)
│   ├── Instagram carousel (visual framework)
│   └── Short-form video (hook + one insight)
├── Email derivatives (2-3 pieces)
│   ├── Newsletter feature (summary + link)
│   └── Nurture email (one insight + CTA)
└── Micro-content (ongoing)
    ├── Quote graphics (data points)
    ├── Infographic (framework visualization)
    └── Community replies (use insights to answer questions)
```

## Adaptation Rules

### What Changes Per Channel

| Element | Blog → LinkedIn | Blog → Twitter | Blog → Carousel | Blog → Email |
|---------|----------------|---------------|-----------------|-------------|
| **Length** | 1-3 paragraphs | 280 char per tweet | 8-10 slides | 2-3 paragraphs |
| **Hook** | Professional insight | Provocative/contrarian | Visual + bold text | Personal/direct |
| **Tone** | Authoritative | Conversational | Visual-first | Intimate |
| **CTA** | Comment/discuss | Retweet/reply | Save/share | Click through |
| **Format** | Text, maybe image | Thread or single | Slides with design | Plain text works |

### What Stays the Same

- **Core insight** — the fundamental point doesn't change
- **Evidence** — same data/quotes, just reformatted
- **Audience pain** — same pain point, different framing
- **Brand voice** — consistent personality across channels

### What Must NOT Be the Same

- Don't copy-paste the same text across channels — each platform has native patterns
- Don't use the same hook twice — adapt it to each channel's attention mechanics
- Don't use the same CTA — match it to what's natural on that platform

## Hero → Derivative Workflow

### Step 1: Choose the Hero

Hero content is your highest-effort, deepest piece. Usually:
- Blog post / guide (2,000+ words)
- Video (10+ minutes)
- Research report
- Podcast episode
- Webinar

### Step 2: Extract Derivatives

From one hero piece, extract:

| Derivative Type | What to Extract | Quantity |
|----------------|----------------|----------|
| **Key insight** | The single most surprising finding | 1 per platform |
| **Framework** | Any model, matrix, or process | 1 carousel + 1 infographic |
| **Data point** | Specific numbers or stats | 2-3 quote graphics |
| **Hot take** | Contrarian or provocative angle | 1 thread or post |
| **How-to** | Actionable steps from the piece | 1 thread or carousel |
| **Story/example** | Case study or anecdote | 1 post |
| **Question** | Open-ended question from the topic | 1 engagement post |

### Step 3: Adapt and Schedule

- Spread derivatives across 1-2 weeks (don't publish all at once)
- Lead with the strongest derivative, not the hero
- Cross-link: derivatives reference the hero; hero references can link back
- Track which derivative format drives the most engagement back to the hero

---

## Repurposing Anti-Patterns

**Copy-paste with minor edits** — Posting the same paragraph on LinkedIn, Twitter, and email with only length changes. Each platform needs a native hook, native format, and native CTA.

**Hero-first distribution** — Publishing the long-form piece first, then scrambling for social posts. Reverse it: tease insights via social derivatives for a week, THEN release the hero piece. Builds anticipation and primes the audience.

**One-and-done derivatives** — Creating one LinkedIn post from a 3,000-word guide. A single hero piece should produce 5-10 derivatives minimum across channels and time.

**Ignoring format conventions** — Writing a Twitter thread like a blog excerpt, or a carousel like a long caption. Each format has native patterns — respect them or the content underperforms regardless of quality.

## Cascade Planning Template

```markdown
## Hero: [Title]
**Type:** Blog / Video / Report / Webinar
**Publish date:** [date]

### Extracted Assets
1. Key insight: "[one sentence]"
2. Framework: [name of model/matrix]
3. Data points: [list specific numbers]
4. Hot take: "[contrarian angle]"
5. Story: [case study or example]

### Derivative Schedule
| Day | Platform | Format | Asset Used | Status |
|-----|----------|--------|-----------|--------|
| D-3 | Twitter | Hot take post | #4 | Draft |
| D-1 | LinkedIn | Insight post | #1 | Draft |
| D0 | Blog | Hero published | All | Draft |
| D+1 | LinkedIn | Carousel | #2 | Draft |
| D+2 | Twitter | Thread | #3 + #5 | Draft |
| D+3 | Email | Newsletter feature | #1 + link | Draft |
| D+5 | Instagram | Carousel | #2 visual | Draft |
```
