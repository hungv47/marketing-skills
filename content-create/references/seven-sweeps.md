# Seven Sweeps Editing Method

A structured multi-pass editing workflow where each pass focuses on exactly one dimension. The sequence matters — edit in this order.

---

## Why Sequence Matters

Editing clarity before emotion prevents adding emotion to unclear copy. Editing specificity before risk removal prevents removing friction that's actually necessary for understanding. Each sweep builds on the previous one.

---

## The Seven Sweeps

### Sweep 1: Clarity

**Focus:** Can every sentence be understood on first read?

- [ ] Remove or define jargon the audience wouldn't know
- [ ] Break sentences >25 words into shorter ones
- [ ] Replace passive voice with active ("was improved by" → "improved")
- [ ] Ensure every pronoun has a clear referent ("it," "this," "they" — what?)
- [ ] Paragraphs are 2-4 sentences for web (shorter for social)
- [ ] Cut "very," "really," "just," "actually," "utilize" (replace with nothing or a stronger word)

**Test:** Read each sentence in isolation. Does it stand alone?

### Sweep 2: Voice & Tone

**Focus:** Does it sound like the brand, consistently?

- [ ] Read the entire piece aloud — where do you stumble?
- [ ] Check tone consistency (don't switch from casual to formal mid-piece)
- [ ] Verify voice matches the brand's 3 adjectives (from product-context.md)
- [ ] Remove corporate-speak that snuck in ("leverage," "synergy," "ecosystem")
- [ ] Check for AI copy slop ("Unlock the power of," "In today's fast-paced world")

**Test:** Would the persona recognize this as written by a human who understands them?

### Sweep 3: So What?

**Focus:** Every claim must answer "why should I care?"

- [ ] For each statement, ask: "So what? What does this mean for the reader?"
- [ ] If the answer is "nothing specific," either add the implication or cut
- [ ] Features must connect to outcomes ("Real-time sync" → "Never ask 'did you see my update?' again")
- [ ] Use the "which means..." bridge: "[Feature], which means [benefit for them]"

**Test:** If the reader said "so what?" after each paragraph, would you have an answer?

### Sweep 4: Prove It

**Focus:** Every assertion needs evidence.

- [ ] Claims without evidence: add a testimonial, statistic, or third-party reference
- [ ] Replace vague social proof ("trusted by many") with specific ("used by 3,000 teams including Stripe")
- [ ] Check: are numbers cited? Are sources named?
- [ ] Flag any claim a skeptical reader would challenge — add proof or soften

**Test:** Would a skeptical reader accept each claim without additional research?

### Sweep 5: Specificity

**Focus:** Replace every abstraction with a concrete detail.

- [ ] Apply the Zoom-In Technique: abstract → ask "what do I actually mean?" → concrete
- [ ] Replace adjectives with facts ("fast" → "loads in 200ms")
- [ ] Replace categories with examples ("multiple integrations" → "Slack, Notion, and Linear")
- [ ] Add numbers wherever possible ("save time" → "save 4 hours/week")
- [ ] Run the Falsifiability check: can this be proven true or false?

**Test:** Can the reader picture it? If they close their eyes, does an image form?

### Sweep 6: Heightened Emotion

**Focus:** Make the reader feel something.

- [ ] Strengthen "before" states — make the pain vivid and specific
- [ ] Add sensory language where appropriate (what does the pain look/feel like?)
- [ ] Check emotional arc: does the piece move from pain → hope → action?
- [ ] Verify emotional triggers match ICP emotional drivers (from icp-research)
- [ ] Don't manufacture emotion — amplify what's already there from the evidence

**Test:** Would someone in this situation read the first line and think "that's exactly how it feels"?

### Sweep 7: Zero Risk

**Focus:** Remove every barrier to the desired action.

- [ ] Guarantee visible near every CTA
- [ ] Objections addressed before the ask
- [ ] Risk-reversal language present ("Cancel anytime," "No credit card required")
- [ ] CTA is specific about what happens next (not vague "Get Started")
- [ ] No surprise commitments hidden in the flow
- [ ] Exit is graceful — don't guilt-trip the reader for not converting

**Test:** What's the worst thing that could happen if they click? If the answer isn't "nothing," add a risk-reverser.

---

## Quick-Pass Rules (for non-key lines)

Not every line needs all seven sweeps. For body copy (not key lines):

- Sentences under 25 words
- Paragraphs 2-4 sentences for web
- Cut "very/really/just/actually"
- Replace "utilize" with "use"
- One idea per paragraph

Key lines (hooks, headlines, CTAs, taglines, subject lines) always get the full seven sweeps.

---

## Common Problems → Sweep Solutions

| Problem | Which Sweep Fixes It |
|---------|---------------------|
| "Wall of features" copy | Sweep 3 (So What) — bridge each to a benefit |
| Copy sounds like AI wrote it | Sweep 2 (Voice) — read aloud, add personality |
| Reader doesn't trust the claims | Sweep 4 (Prove It) — add evidence |
| Copy is vague and forgettable | Sweep 5 (Specificity) — zoom in on details |
| Copy is logical but flat | Sweep 6 (Emotion) — add vivid "before" states |
| Readers don't click the CTA | Sweep 7 (Zero Risk) — remove barriers |
